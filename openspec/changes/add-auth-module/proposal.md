# 变更：添加认证和注册模块

## 为什么

应用程序目前缺少用户认证系统，这对于个性化用户体验和保护用户数据至关重要。我们需要实现完整的登录和注册流程，以支持用户账户管理和安全的 API 访问。

## 变更内容

- 添加带有用户名/密码表单的用户注册页面
- 添加带有用户名/密码认证的用户登录页面
- 实现基于 token 的认证，包含会话 ID (SID) 管理
- 添加使用已存储 token 的自动登录功能
- 实现请求拦截器，在所有 API 请求中包含 SID
- 在本地存储中添加安全的 token 存储
- 实现启动时认证检查以确定登录状态
- 添加导航逻辑，将未认证用户重定向到登录页面
- 为登录和注册页面设计现代、美观的 UI

## 影响

- 受影响的规范：
  - **auth** (新功能) - 用户认证和会话管理
- 受影响的代码：
  - `src/pages/` - 添加登录和注册页面组件
  - `src/services/` - 添加认证服务和 API 集成
  - `src/utils/` - 添加 SID 注入的请求拦截器
  - `src/app.ts` - 添加启动认证检查逻辑
  - `src/store/` - 添加认证状态管理
